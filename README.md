# 🛡️ マルチテナントRAG

マルチテナントRAGの **データ分離・AIあり/なし比較・実行コスト可視化・監査ログ** を確認するPoCです。

## 何をするPoCか

複数部署・複数テナントで同じAI/RAG基盤を使う場合に、  
ログインユーザーの `tenant_id` に応じて検索対象を制限し、他テナント文書がAI回答に混ざらないことを確認します。

さらに、同じ質問に対して以下を比較できます。

- **AIなし**：検索結果だけを表示する
- **AIあり**：検索結果をもとに回答文を生成する

AIを使うと便利になりますが、従量課金・権限制御・監査ログが必要になります。  
このPoCでは、その違いを画面上で確認できます。

## 主な確認ポイント

- ユーザーごとに参照できる文書が変わる
- 他テナント文書を検索対象から除外する
- 「情報がないから回答できません」と「権限がないから回答できません」を分けて表示する
- 権限なし・情報なしの場合はAIを呼ばず、無駄なコストを発生させない
- AIあり/なしの違いを比較できる
- 実行ごとの入力/出力トークン数と概算円コストを表示する
- 監査ログをSQLiteに保存する
- 他テナント候補のブロック件数をログに残す

## モデル選択

画面左側で、利用するプロバイダーとモデルを選択できます。

- Anthropic
- OpenAI

利用可能なモデルIDはAPI提供状況により変わるため、画面上で編集できるようにしています。

## 安全設計の考え方

このPoCでは、AIに「他部署の情報を見ないで」と依頼するのではなく、  
**AIに渡す前の検索段階で tenant_id により検索対象を制限**します。

本番では「情報なし」と「権限なし」を同じ文言にする選択肢もあります。  
このPoCでは、制御内容を分かりやすくするため、あえて画面上で理由を分けて表示します。
